# ADR-008: iOS 확장은 모션 기반 수집 + 무료 케이블 설치로 한다 (착수 보류)

- **상태**: 확정 (2026-07-18) — 방침만 확정, 코드 착수는 백로그. 착수 조건 = 검증용 iPhone 확보
- **결정**: iOS 확장은 실행 가능으로 확정한다. 수집은 iOS 기본 **모션 기반**(정지 중 수집 포기, 배터리 우선), 설치는 **무료 경로**(Mac + USB 케이블 직접 설치)만 쓴다.
- **대가**: iOS의 점 밀도는 Android보다 훨씬 낮아지고, 무료 서명은 7일 만료라 상시 설치 유지가 안 된다.

## 배경

앱은 현재 Android 전용이다(`ios/` 플랫폼 미추가). iOS로도 올려보고 싶은데, 착수 전에 세 가지가 불확실했다: ① 기술적으로 막히는 게 있는가 ② 배터리를 많이 먹지 않고 가능한가(많이 먹으면 최악) ③ 완전 무료로 가능한가. 이 문서는 세 질문의 답을 확정해 두고 실행을 뒤로 미루는 기록이다 — 나중에 착수할 때 다시 조사하지 않기 위해.

조사 결과: 사용 중인 플러그인 6개 전부 iOS 지원(핵심 수집 SDK는 iOS 무료), 무료 Apple ID로 실기기 설치 가능.

## 결정 ① 수집 정책 — heartbeat 포기, 이동 중에만 수집

현재 Android 수집은 두 축이다: 이동 중엔 위치 스트림, **정지 중엔 heartbeat**(1분마다 위치 1개)로 1분 간격을 유지한다. 그런데 iOS는 정지 상태의 앱을 OS가 재워버리는(suspend) 구조라 잠든 앱에는 heartbeat가 오지 않는다.

| 대안 | 배터리 | 판단 |
|---|---|---|
| **모션 기반 수집 (채택)** — 이동이 감지되면 위치를 받고, 정지하면 GPS를 끄고 앱을 재움. 밀도는 시간이 아니라 이동 거리(`distanceFilter`, 50m 안팎)로 결정 | iOS 표준 저전력 경로 | 정지 중 공백은 기존 병합 규칙이 흡수(아래) → 손실 없이 배터리 목표 달성 |
| `preventSuspend: true`로 1분 간격 유지 | 앱을 상시 깨워둠 — 소모 큼 | "배터리 많이 먹으면 최악" 기준으로 탈락 |

### 정지 중 포인트가 0개인데 체류 통계가 성립하는 근거

관건은 "8시간 사무실에 있는 동안 점이 하나도 안 찍혀도 8시간 체류로 잡히는가"였다. **성립한다.** 근거는 체류 판정에 이미 있는 수집 공백 병합 규칙([ADR-005](005-체류판정-이탈유예-공백분리.md))이다.

- 체류 시간은 점 개수가 아니라 클러스터의 첫 점~마지막 점 시간 간격으로 계산한다.
- 같은 반경 안의 점 사이에 큰 공백(기본 3시간 초과)이 있어도, 쌓인 쪽이 아직 유효 체류(10분) 미달이면 끊지 않고 병합한다.
- iOS 시나리오를 넣으면: 도착 직전 이동 점 몇 개(10분 미달) → 8시간 공백 → 출발 점. 공백이 3시간을 넘지만 쌓인 쪽이 미달이라 병합 분기를 타고, 도착~출발이 하나의 체류로 이어진다.

즉 Android용 방어 로직이 iOS의 희소 수집 패턴을 그대로 받아준다. 통계·체류 로직 재설계는 불필요하다.

## 결정 ② 설치 경로 — 무료(케이블 직접 설치)만

전제 지식: iOS는 Android의 APK와 달리 앱 파일(.ipa)을 받아서 바로 설치하는 경로가 없다. 모든 앱은 Apple이 발급한 서명에 묶여야 실행되고, 그 서명에는 실행 허용 기기 목록이 박힌다.

| 경로 | 비용 | 가능 여부·비고 |
|---|---|---|
| **Xcode + USB 케이블 설치 (채택)** | 0원 | 무료 Apple ID로 서명 → 연결한 기기에 설치. 7일 만료(재설치 필요), 기기당 앱 3개 제한, 개발자 모드 필요 |
| AltStore/Sideloadly 류 | 0원 | 무료 서명의 자동화 도구일 뿐 — 똑같이 7일 만료 + PC에 도우미 상주 |
| TestFlight | $99/년 | 링크 설치·90일 유지. "남에게 나눠주는 배포"가 필요해지면 그때 재검토 |
| Ad Hoc(.ipa 전달) | $99/년 | 기기 UDID 사전 등록 필요. TestFlight 하위호환이라 검토 가치 없음 |
| App Store | $99/년 | 앱이 무료여도 개발자 프로그램 등록비는 유료. 수익화 안 하므로 불필요 |

핵심 확인: **개발자 프로그램($99/년)은 "스토어에 올릴 자격"이지 "기기에서 돌릴 자격"이 아니다.** 내 빌드를 실기기에 올려 쓰는 것까지는 완전 무료다.

## 착수 시 변경 범위 — 최소 변경

1. **`src/features/collector/useCollector.ts`** — `ready()` 설정에 iOS 분기: heartbeat 미사용, `distanceFilter` 부여. Android 전용 옵션(포그라운드 알림, `locationUpdateInterval`)은 iOS에서 자동 무시되므로 방치.
2. **iOS 프로젝트 생성물** — `npx cap add ios` 후 Info.plist에 위치 권한 문구 3종 + 백그라운드 위치 모드 선언.

UI·DB·통계·체류 판정은 무수정. 권한 상태 코드(`AuthorizationStatus`)는 양 플랫폼 공용이라 재사용된다.

## 한계 (수용)

- 정지 중 수집을 포기하므로 iOS의 점 밀도는 Android보다 훨씬 낮다. 체류는 성립하지만, 점 밀도에 기대는 통계 표현(히트맵 등)은 iOS 데이터에서 얇게 나올 수 있다.
- `stopOnTerminate: false`·`startOnBoot`는 iOS에서도 동작하지만 메커니즘이 다르다(큰 위치 변화 시 OS가 앱을 재기동). 보장 수준은 실기기 관찰로만 확인 가능.
- 무료 서명 7일 만료 탓에 상시 설치 유지가 안 된다 — "올려보는 경험" 목적에는 충분하다.
