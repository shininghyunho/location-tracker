# ADR-003: 웹 스택은 Vite+React+TS / SQLite / Leaflet

- **상태**: 확정 (2026-07-17)
- **결정**: 프런트 = **Vite + React 18 + TypeScript**(+ react-query, Tailwind) · 저장소 = **SQLite**(`@capacitor-community/sqlite`) · 지도 = **Leaflet + OSM 타일**, 단 지도 SDK는 훅+컨트롤러 래퍼로 감싼다.
- **대가**: SSR·GraphQL 같은 서버 전제 스택 경험은 이 프로젝트에서 얻지 못한다. 상용 지도 SDK 실사용 경험도 제외 — 래퍼 패턴으로 벤더 교체 여지만 남긴다.

## 배경

웹뷰 래퍼 = Capacitor([ADR-002](002-웹뷰-래퍼-Capacitor.md))를 전제로, 그 위에 올릴 웹 스택 세 가지(프레임워크·저장소·지도)를 정해야 했다. 선정 기준은 "최신·최적"이 아니라 두 가지다.

- **훈련 효과** — 이 프로젝트는 React 생태계(컴포넌트·훅·데이터 패칭·지도 래핑 패턴) 훈련을 겸한다. 매일 만지게 될 패턴이 실제 웹 개발 근육으로 전이되는 스택을 고른다.
- **서버 없음** — 모든 데이터는 로컬에만 있다. 서버가 있어야 성립하는 스택(SSR, GraphQL, 팀 규모 전역 상태)은 이름값이 아무리 높아도 이 앱에선 배울 것도 쓸 이유도 없다.

## 프런트 프레임워크: Vite + React 18 + TS

| 후보 | 판정 | 이유 |
|---|---|---|
| **Vite + React SPA** | ✅ 채택 | 정적 빌드 결과물을 Capacitor 웹뷰가 그대로 로드 — 마찰 없음. React·TS·react-query·Tailwind 훈련 효과 최대 |
| Next.js | ❌ 기각 | Next의 학습 가치는 SSR·서버 라우팅인데 우리는 서버가 없어 정적 export로만 쓰게 됨 → 이름만 남고 배우는 건 React뿐. Capacitor와의 정적 export 설정 마찰만 추가 |

같은 이유로 GraphQL(API 서버 전제)과 Redux 급 전역 상태(이 화면 수에선 react-query + 로컬 state로 충분)도 배제했다.

## 로컬 저장소: SQLite

| 후보 | 판정 | 이유 |
|---|---|---|
| **SQLite** (`@capacitor-community/sqlite`) | ✅ 채택 | points가 수십만 행까지 자라고(1분 간격 수집 시 연 50만+), 핵심 화면이 "장소별 누적 체류시간" 같은 **집계**다 → SQL의 GROUP BY가 정답 |
| IndexedDB (웹 표준) | ❌ 기각 | 집계 쿼리가 없어 JS 루프로 직접 구현해야 함. 수십만 행 집계에서 성능·코드량 모두 불리 |

## 지도: Leaflet + OSM 타일 (+ 래퍼 패턴)

| 후보 | 판정 | 이유 |
|---|---|---|
| **Leaflet + OSM 타일** | ✅ 채택 | 무료·API 키 불필요·출처 제한 없음. MVP 완성 우선 원칙에 부합 |
| 상용 지도 SDK (네이버·구글 등) | ❌ 기각 | 국내 지도 품질은 낫지만, 등록된 도메인에서만 동작하는 키 정책이라 Capacitor 웹뷰 출처(`capacitor://localhost`) 등록이 껄끄럽고, 키·쿼터 관리도 개인 프로젝트엔 마찰 |

단, 지도 SDK를 **훅+컨트롤러 래퍼로 감싸는 구조**(`features/map`)를 채택한다. 컴포넌트가 Leaflet API를 직접 부르지 않게 격리해 두면, 나중에 상용 SDK로 벤더 교체를 원할 때 어댑터 한 곳만 수정하면 된다.

## 결과

**얻는 것**

- UI 개발 경험이 React·훅·react-query·Tailwind·지도 래퍼 패턴 훈련으로 직접 전이된다.
- Capacitor 정적 빌드 파이프라인과 마찰이 없는 조합이라 MVP 속도가 빠르다.
- 수십만 행 points 집계를 SQL로 처리해 통계 화면 구현이 단순해진다.

**잃는 것 (감수)**

- SSR·GraphQL·대형 전역 상태 경험은 이 프로젝트로 얻지 못한다 → 별도 학습으로 커버.
- 상용 지도 SDK 실사용 경험이 빠진다 → 래퍼 패턴으로 절반 보전, 필요 시 추후 벤더 교체.
- SQLite 플러그인이 커뮤니티 관리 품질이다 → 수집기 PoC에서 저장까지 묶어 검증 완료. 이후 실사용에서 수십만 행 적재·집계까지 정상 동작 확인.
