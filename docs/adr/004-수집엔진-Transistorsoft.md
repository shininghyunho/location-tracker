# ADR-004: 수집 엔진은 Transistorsoft SDK를 쓴다 (커뮤니티 플러그인 교체)

- **상태**: 확정 (2026-07-17)
- **결정**: 백그라운드 GPS 수집 플러그인을 `@capacitor-community/background-geolocation` → `@transistorsoft/capacitor-background-geolocation` v9로 교체한다. debug 빌드는 라이선스 없이 전 기능 무료.
- **대가**: 내부가 블랙박스(학습 효과↓). 스토어 배포용 Android release 빌드를 만들 때는 라이선스(약 $349) 필요 — 지금은 debug APK 사이드로드 운영이라 비용 0, release 시점에 다시 결정.

## 배경

앱을 스와이프로 완전히 끄거나 폰을 재부팅하면 수집이 멈췄다. 요구는 "구글맵처럼 앱이 꺼져도 계속 수집"인데, 커뮤니티 플러그인으로는 두 겹의 문제가 있었다.

1. **플러그인이 일부러 죽는다** — 서비스 코드에 "앱 본체가 죽으면 서비스도 즉시 종료"가 명시돼 있다(`onUnbind()`에서 `stopSelf()` — 앱 없이 서비스만 남으면 크래시 나는 업스트림 이슈의 회피책). 재부팅 후 자동 시작 코드도 없다.
2. **구조적으로 저장할 주체가 없다** — 위치를 JS(웹뷰)가 받아 DB에 쓰는 구조인데, 앱이 죽으면 웹뷰도 죽는다. 서비스만 살려도 소용없고, **네이티브 쪽에서 저장**해야 한다.

참고: 구글맵의 타임라인 수집은 앱이 아니라 Google Play Services(OS 수준 시스템 구성요소)가 한다. 일반 앱은 그 수준엔 못 가지만, "종료·재부팅 생존"까지는 포그라운드 서비스 + 네이티브 저장으로 도달 가능하다.

## 검토한 대안

| 대안 | 장점 | 단점 | 판단 |
|---|---|---|---|
| **Transistorsoft 교체 (채택)** | 이 문제만 10년+ 판 검증품. 종료·재부팅 생존 + 네이티브 SQLite 저장 내장. Capacitor 8 전용 v9 존재 | 내부 블랙박스. Android release에 라이선스 약 $349 | 수집 안정성이 최우선 요구라 신뢰성 우선 |
| 커뮤니티 플러그인 포크(자작) | 무료, 전 과정 학습 | 제조사 절전 킬 등 신뢰성 리스크를 직접 짐. 구현+검증 1~2일, 계속 유지보수 | 학습 가치는 크지만 실사용 신뢰성이 우선 |
| 현상 유지(운영으로 버티기) | 작업 0 | 스와이프 한 번에 수집 중단 | 요구사항 미달 |

## 새 데이터 흐름

```
[앱이 죽어 있어도 계속]                      [앱을 열었을 때]
네이티브 TrackingService                    JS가 getLocations()로 회수
  → 플러그인 자체 SQLite에 적재      →       → 앱 points 테이블에 insert
    (transistor_location_manager)            → 옮긴 레코드만 uuid로 삭제
                                             → 체류판정~백업 파이프라인은 그대로
```

- 설정: `stopOnTerminate: false`(종료 생존) · `startOnBoot: true`(재부팅 생존) · 1분 간격 + 정지 시엔 모션 감지로 GPS를 쉬고 heartbeat(앱 활성 시)로 보완.
- 같은 GPS 픽스가 레코드 2개로 올 때가 있어 points에 `(ts, source)` 유니크 인덱스 + `INSERT OR IGNORE`로 멱등 처리.
- **프라이버시 원칙 유지**: 이 SDK에는 서버 업로드(HTTP) 기능이 있지만 설정하지 않았다 — 위치는 여전히 기기 밖으로 나가지 않는다.

## 한계 (수용)

- 정지 중 + 앱 종료 상태에선 점이 안 쌓인다(모션 감지가 GPS를 쉼). 떠나는 순간의 motionchange 레코드가 같은 자리에 찍혀 체류 종료 시각은 복원된다 — 체류 판정에는 영향 없음.
- 재부팅 자동 시작(`startOnBoot`)은 실기기 검증 완료(2026-07-17): BOOT_COMPLETED → 플러그인 자가 기동·위치 픽스·정지 감시 재개, 앱을 열면 heartbeat 1분 수집까지 재개. 주의 — 부팅 후 첫 잠금해제 전엔 BOOT_COMPLETED가 안 오고, 정지 상태에선 플러그인이 점을 안 쌓으며(설계), heartbeat 픽스는 JS가 살아 있을 때만 찍힌다.
