# ADR-005: 체류 판정 보정 — 이탈 유예 + 정확도 전처리 + 수집 공백 분리

- **상태**: 확정 (2026-07-18)
- **결정**: 실시간 체류판정(`detectStays`)에서 ① 클러스터를 닫는 기준을 "반경 밖 점 1개"에서 "반경 밖에 **연속 `graceMs` 이상** 머묾"으로 바꾸고, ② 판정 전에 정확도 나쁜 점(`accuracy_m > maxAccuracyM`)을 제외하며, ③ 수집 공백(> `maxGapMs`)은 쌓인 쪽이 유효 체류일 때만 분리한다.
- **대가**: 판정 로직에 상태(out-run 추적)가 추가되고, `graceMs`에 살짝 못 미치는 실제 짧은 이동은 체류로 흡수될 수 있다(설정값이라 조정 가능).

## 배경 — 두 방향의 오판

체류판정은 시간순 점을 훑으며 중심 반경 안이면 같은 클러스터, 벗어나면 클러스터를 닫는다. 여기에 대칭인 문제 둘이 있었다.

- **쪼개짐**: 닫는 판정이 **점 1개 기준**이라, 잠깐 자리를 비우거나(화장실 등) GPS가 한 번 튀면 체류가 거기서 끊긴다. 되돌아오면 새 클러스터가 시작돼, 하나의 연속 체류가 여러 조각이 된다.
- **부풀음**: 반대로 반경 안이기만 하면 직전 점과의 **시간차를 안 봤다.** 강제종료·배터리 등으로 수집이 몇 시간 끊겼다가 같은 자리에서 재개되면, 그 공백이 하나의 연속 체류 시간으로 부푼다.

## 실데이터가 방향을 정했다

기기 DB(2026-07-18)에서 수집기 점 347개를 뽑아 분석했다.

- 정확도가 **전부 30m 이하**(스파이크 0건) → GPS 오차가 쪼개짐의 주원인이 아니다. 정확도 필터 단독으로는 못 잡는다.
- 반면 같은 장소 라벨의 점들이 여러 방문에 걸쳐 대각선 약 165m로 산포 → 실제 원인은 **"잠깐 반경을 벗어나는 실제 위치 이동"**(짧은 외출, 넓은 장소 안 이동)이다.

그래서 시간 기반 이탈 유예(①)를 핵심 해법으로, 정확도 필터(②)는 값싼 보험으로 겸용한다.

## 채택한 규칙

```
각 점 p (클러스터 열림, 중심 c):
  반경 안  → out-run 리셋. 단 직전 점과의 공백 gap을 본다:
      ├ gap ≤ maxGapMs                          → 클러스터에 추가
      ├ gap > maxGapMs, 쌓인 체류 ≥ minDuration  → 여기서 끊고(유효 stay 확정) p로 새 클러스터
      └ gap > maxGapMs, 쌓인 체류 < minDuration  → 병합(끊지 않음) — 밤샘 체류 소실 방지
  반경 밖  → out-run 이어감
      ├ out-run < graceMs  → p 건너뜀(허용 blip). 클러스터 유지, 중심 오염 없음
      └ out-run ≥ graceMs  → 이탈 확정. 클러스터 닫고 새 클러스터 시작
```

공백 분리(③)에 조건이 붙은 이유: 이 수집 모델은 정지 상태에선 점을 거의 안 찍는다([ADR-004](004-수집엔진-Transistorsoft.md) 한계). "공백이 크면 무조건 끊기"로 하면 점이 드문 밤샘 집체류가 공백마다 끊겨 양쪽 조각이 둘 다 최소 체류시간 미달이 되고, **체류가 통째로 사라진다.** 그래서 쌓인 쪽이 이미 유효 체류일 때만 분리한다.

## 튜닝값 (하드코딩 금지 — `StayParams`)

| 값 | 기본 | 의미·근거 |
|---|---|---|
| `graceMs` | 5분 | 몇 분까지 자리를 비워도 같은 장소로 볼지. 화장실·짧은 볼일 흡수, 그 이상은 별도 이동으로 분리 |
| `maxAccuracyM` | 100m | 판정 반경보다 부정확한 점은 신뢰 불가. 실데이터(≤30m)에선 정상 점이 하나도 안 잘림. `accuracy_m`이 null인 점(import)은 유지 |
| `maxGapMs` | 3h | 이 시간 넘게 점이 없으면 수집 블랙아웃으로 본다. 이동 집계의 gap 컷과 같은 정의로 맞춰 체류/이동 기준을 일치 |

## 검토한 대안

| 대안 | 장점 | 단점 | 판단 |
|---|---|---|---|
| **이탈 유예 + 정확도 필터 (채택)** | 짧은 blip을 시간 기준으로 흡수, 긴 이탈은 그대로 분리. 중심 오염 없음 | 판정 로직에 상태 하나 추가 | 원인(시간성 이탈)에 직접 대응 |
| 사후 병합(인접 클러스터 중심이 가까우면 합치기) | 판정 로직을 안 건드림 | 이미 저장된 stay를 다시 꺼내 삭제·갱신해야 해 증분 커서 경계와 충돌. 왕복 외출과 실제 이동을 중심만으로 구분하기 약함 | 커서 경계·오병합 리스크로 탈락 |
| 정확도 필터 단독 | 가장 단순 | 실데이터가 증명하듯 스파이크가 주원인이 아님 | 보조로만 채택 |

## 결과

- 변경은 실시간 판정 한 파일(`detectStays`)에 국한된다. import stay는 완성돼 들어오므로 무관.
- 증분·멱등 유지 — 결과는 점 시퀀스로만 결정되므로 같은 구간을 재처리해도 동일하다. 확정 시점만 grace만큼 뒤로 밀린다.
- 검증: 합성 시나리오 검산(이탈 유예 6/6, 공백 분리 5/5) + 실데이터 회귀(347점이 여전히 1클러스터) + 회귀 17/17.
